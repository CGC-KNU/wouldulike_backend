# 스탬프/리워드 백엔드 확인 결과

## 1. 고니식탁 (restaurant_id: 30) – rewards 누락 및 쿠폰 미발급

### 원인
**고니식탁은 스탬프 import 대상에서 제외되어 있었습니다.**  
`import_stamp_rewards_from_csv`의 EXCLUDE 목록에 포함되어 커스텀 StampRewardRule이 없었고, 시드 마이그레이션 기본값(5, 10)만 적용된 상태였습니다.

### 실제 혜택 (적용됨)
- **3개**: 음료 서비스
- **5개**: 계란말이 서비스
- **10개**: 찌개 1인분 서비스

### 조치
고니식탁을 스탬프 import에 추가하여 3, 5, 10개 혜택이 반영되도록 수정했습니다.

### API 동작
- **all API**: `rewards` 배열 포함 (각 항목에 `rewards`, `notes` 포함)
- **단건 API**: `rewards` 포함
- **쿠폰 발급**: 3·5·10개 도달 시 `reward_coupon_code`, `reward_coupons` 반환

## 2. 다이와스시 (restaurant_id: 56) – 3개 혜택 표시

### 실제 혜택 (적용됨)
- **5개**: 타코야끼 10개
- **10개**: 만원 할인

프론트에서 “5개에만 혜택”이라고 보고되었으나, general.csv에는 **3개, 10개**에 혜택이 있습니다.  
계약/CSV가 변경되었다면 5개만 혜택으로 두는 설정으로 수정이 필요합니다.

## 3. API 응답 구조

### GET /api/coupons/stamps/my/?restaurant_id={id}
```json
{
  "current": 2,
  "target": 10,
  "rewards": [
    { "stamps": 3, "title": "...", "subtitle": "", "notes": "", "benefit": {...}, "coupon_type_code": "STAMP_REWARD_3" },
    ...
  ],
  "notes": "",
  "updated_at": "..."
}
```

### GET /api/coupons/stamps/my/all/
```json
{
  "results": [
    {
      "restaurant_id": 30,
      "current": 0,
      "target": 10,
      "rewards": [...],
      "notes": "",
      "updated_at": null
    },
    ...
  ]
}
```

- all API와 단건 API 모두 동일한 `rewards` 구조 사용
- THRESHOLD: `stamps`, `title`, `subtitle`, `notes`, `benefit`, `coupon_type_code`
- VISIT: `visit_range`, `min_visit`, `max_visit`, `title`, ...

### POST /api/coupons/stamps/add/
쿠폰 발급 시:
```json
{
  "ok": true,
  "current": 3,
  "target": 10,
  "reward_coupon_code": "ABC123",
  "reward_coupon_codes": ["ABC123"],
  "reward_coupons": [
    { "threshold": 3, "coupon_code": "ABC123", "coupon_type": "STAMP_REWARD_3" }
  ]
}
```
